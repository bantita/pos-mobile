# การออกแบบประเภทร้านค้า POS Mobile

> เอกสารออกแบบ v2.0 — ปรับโครงสร้างใหม่ทั้งหมด

---

## 1. ภาพรวมโครงสร้าง

ระบบแบ่งร้านค้าเป็น **2 มิติ** ที่เลือกอิสระจากกัน:

### มิติที่ 1: ประเภทธุรกิจ (Business Type)

| ประเภท | คำอธิบาย | ตัวอย่าง |
|--------|----------|----------|
| **SERVICE** | ร้านบริการ — สินค้าเป็นบริการ ต้องเลือกพนักงานบริการทุกครั้ง | ร้านตัดผม, สปา, นวด, เสริมสวย, คลินิก |
| **RETAIL** | ร้านค้าปลีก/ค้าส่ง — สินค้าทั่วไป ขายแบบปกติ | ร้านสะดวกซื้อ, ร้านขายส่ง, ร้านอาหาร, คาเฟ่ |

### มิติที่ 2: ขนาดธุรกิจ (Business Scale)

| ระดับ | คำอธิบาย | โครงสร้าง |
|-------|----------|-----------|
| **BUSINESS** | ร้านเดียว หลายจุดขาย | 1 ร้าน → 1-N จุดขาย (POS) |
| **ENTERPRISE** | หลายสาขา หลายจุดขาย | 1 บริษัท → 1-N สาขา → 1-N จุดขาย/สาขา |

### ผลลัพธ์ 4 แบบที่เป็นไปได้

| # | ประเภท + ระดับ | ตัวอย่าง | โครงสร้าง |
|---|----------------|----------|-----------|
| 1 | SERVICE + BUSINESS | ร้านตัดผม 1 สาขา มี 3 เก้าอี้ | ร้าน → 3 จุดขาย |
| 2 | SERVICE + ENTERPRISE | เครือสปา 5 สาขา | บริษัท → 5 สาขา → หลายจุด/สาขา |
| 3 | RETAIL + BUSINESS | ร้านมินิมาร์ท มี 2 แคชเชียร์ | ร้าน → 2 จุดขาย |
| 4 | RETAIL + ENTERPRISE | เครือร้านสะดวกซื้อ 10 สาขา | บริษัท → 10 สาขา → หลายจุด/สาขา |

## 2. โครงสร้างข้อมูล (Data Structure)

### 2.1 Store Config

```typescript
type BusinessType = 'SERVICE' | 'RETAIL';
type BusinessScale = 'BUSINESS' | 'ENTERPRISE';

interface StoreConfig {
  businessType: BusinessType;      // ประเภทธุรกิจ
  businessScale: BusinessScale;    // ขนาดธุรกิจ
  storeName: string;               // ชื่อร้าน/บริษัท
  serviceCharge: ServiceChargeConfig;
}

interface ServiceChargeConfig {
  enabled: boolean;
  mode: 'percentage' | 'fixed';
  value: number;
}
```

### 2.2 โครงสร้างสาขา/จุดขาย

```
[BUSINESS]
  Store (ร้านเดียว)
    └── Terminal 1 (POS 1)
    └── Terminal 2 (POS 2)
    └── Terminal N

[ENTERPRISE]
  Company (บริษัท)
    └── Branch 1 (สาขาหลัก)
    │     └── Terminal 1
    │     └── Terminal 2
    └── Branch 2 (สาขา 2)
    │     └── Terminal 1
    └── Branch N
          └── Terminal 1..N
```

### 2.3 Branch & Terminal

```typescript
interface Company {
  id: string;
  name: string;           // ชื่อบริษัท/เครือ
  taxId?: string;
  address?: string;
}

interface Branch {
  id: string;
  companyId?: string;     // สำหรับ ENTERPRISE
  name: string;           // ชื่อสาขา
  code: string;           // รหัสสาขา เช่น BR01
  address: string;
  phone?: string;
  isHeadquarter: boolean; // เป็นสำนักงานใหญ่?
}

interface Terminal {
  id: string;
  branchId: string;
  name: string;           // ชื่อจุดขาย เช่น POS 1, เก้าอี้ 3
  code: string;           // รหัสจุดขาย เช่น T01
  status: 'active' | 'inactive';
}
```

## 3. ฟีเจอร์ที่แตกต่างตาม Business Type

### SERVICE (ร้านบริการ)

| ฟีเจอร์ | รายละเอียด |
|---------|-----------|
| เลือกพนักงานบริการ | ทุกครั้งที่เพิ่มสินค้าบริการในบิล ต้อง popup เลือกช่าง |
| Service Charge | เปิด/ปิดได้ เก็บ % หรือจำนวนคงที่ต่อบิล |
| สินค้าประเภทบริการ | productType = 'service' — ไม่นับสต๊อก |
| จัดการช่าง/พนักงานบริการ | หน้าจัดการช่าง สถานะว่าง/ไม่ว่าง |
| รายงานรายได้ต่อช่าง | แยกดูยอดขายตามช่างแต่ละคน |
| จองคิว (อนาคต) | ระบบจองคิวล่วงหน้า |

### RETAIL (ร้านค้าปลีก/ค้าส่ง)

| ฟีเจอร์ | รายละเอียด |
|---------|-----------|
| หน่วยนับหลายหน่วย (UOM) | ขวด, แพ็ค, ลัง — ราคาและบาร์โค้ดแยกต่อหน่วย |
| คลังสินค้า | รับ/เบิก/โอน/นับสต๊อก |
| ราคาขายส่ง (อนาคต) | ราคาแบ่งตามจำนวน (Tier Pricing) |
| บาร์โค้ดสแกน | สแกนขายได้เลย ไม่ต้องเลือกช่าง |

## 4. ฟีเจอร์ที่แตกต่างตาม Business Scale

### BUSINESS (ร้านเดียว)

| ฟีเจอร์ | รายละเอียด |
|---------|-----------|
| จุดขายหลายจุด | เพิ่ม Terminal ได้ไม่จำกัด |
| ราคาสินค้า | ราคาเดียว ใช้ทุกจุดขาย |
| ผู้ใช้งาน | กำหนดสิทธิ์ตาม Role (Owner, Manager, Cashier, etc.) |
| รายงาน | รายงานรวมทั้งร้าน + แยกตามจุดขาย |

### ENTERPRISE (หลายสาขา)

| ฟีเจอร์ | รายละเอียด |
|---------|-----------|
| จัดการสาขา | เพิ่ม/แก้ไข/ปิดสาขา |
| จุดขายต่อสาขา | แต่ละสาขามี Terminal ของตัวเอง |
| กำหนดราคาตามสาขา | ราคากลาง vs ราคาเฉพาะสาขา (เอกสารกำหนดราคา) |
| โอนสินค้าระหว่างสาขา | Transfer Stock ข้ามสาขา |
| รายงาน Enterprise | รายงานรวมทุกสาขา + แยกสาขา + เปรียบเทียบสาขา |
| Sync ข้อมูล | Sync สินค้า/ราคา/โปรโมชั่น ระหว่างสาขา |
| สิทธิ์ตามสาขา | กำหนดว่า User เข้าถึงข้อมูลสาขาไหนได้บ้าง |

### ขั้นตอนที่ 3: ตั้งค่าสาขา/จุดขาย

**กรณี BUSINESS:**
- กรอกชื่อจุดขายแรก (เช่น "POS 1" หรือ "เก้าอี้ 1")

**กรณี ENTERPRISE:**
- กรอกชื่อบริษัท
- สร้างสาขาแรก (สาขาหลัก)
- สร้างจุดขายแรกของสาขาแรก

## 7. ฟีเจอร์แยกตามระดับ (Feature Matrix)

| ฟีเจอร์ | BUSINESS | ENTERPRISE |
|---------|----------|-----------|
| จุดขาย POS | ✅ หลายจุด | ✅ หลายจุดต่อสาขา |
| สาขา | ❌ ไม่มี (ร้านเดียว) | ✅ หลายสาขา |
| กำหนดราคาตามสาขา | ❌ ราคาเดียว | ✅ ราคากลาง + ราคาเฉพาะสาขา |
| โอนสินค้าข้ามสาขา | ❌ | ✅ |
| รายงานเปรียบเทียบสาขา | ❌ | ✅ |
| Sync ข้ามสาขา | ❌ | ✅ |
| สิทธิ์ตามสาขา | ❌ (ทุกคนเห็นทั้งหมด) | ✅ กำหนดได้ |
| Import/Export ราคา | ✅ (ราคาเดียว) | ✅ (แยกสาขา) |

| ฟีเจอร์ | SERVICE | RETAIL |
|---------|---------|--------|
| เลือกช่าง/พนักงาน | ✅ popup ทุกครั้ง | ❌ |
| Service Charge | ✅ | ❌ (ปิดไว้) |

| รายงานยอดต่อช่าง | ✅ | ❌ |

## 8. Flow การทำงานหน้า POS ตาม Business Type

### SERVICE Flow:

```
กดสินค้า (type=service)
    │
    ▼
┌─ Popup เลือกช่าง ─┐
│  ช่างเอ (ว่าง)     │
│  ช่างบี (ว่าง)     │
│  ช่างซี (ไม่ว่าง)  │
└────────────────────┘
    │ เลือกช่าง
    ▼
เพิ่มในตะกร้า:
  "ตัดผมชาย"
  "👤 ช่างเอ (สมชาย)"

กดสินค้าเดียวกันอีกครั้ง → popup อีกครั้ง → เลือกช่างคนอื่นได้
```

### RETAIL Flow:

```
กดสินค้า / สแกนบาร์โค้ด
    │
    ▼ (ถ้ามีหลาย UOM)
┌─ Popup เลือกหน่วย ─┐
│  ขวด ฿10            │
│  แพ็ค 6 ฿55         │
│  ลัง 24 ฿200        │
└─────────────────────┘
    │ เลือกหน่วย
    ▼
เพิ่มในตะกร้า:
  "น้ำดื่มสิงห์ 600ml (แพ็ค)"
  "฿55"
```

## 9. การ Migrate ระหว่างระดับ

### BUSINESS → ENTERPRISE (อัปเกรด)

1. ระบบสร้าง "บริษัท" จากข้อมูลร้านปัจจุบัน
2. ร้านปัจจุบัน → เป็น "สาขาหลัก" (isHeadquarter = true)
3. จุดขายเดิม → คงอยู่ภายใต้สาขาหลัก
4. เปิดใช้ฟีเจอร์: จัดการสาขา, กำหนดราคาตามสาขา, โอนสินค้า

### ENTERPRISE → BUSINESS (ดาวน์เกรด)

⚠️ **ข้อจำกัด:** ต้องมีสาขาเดียวเท่านั้น (ลบสาขาอื่นก่อน)

1. ลบข้อมูลบริษัท
2. สาขาที่เหลือ → เป็นร้านหลัก
3. ปิดฟีเจอร์ Enterprise ทั้งหมด

### SERVICE ↔ RETAIL (เปลี่ยนประเภท)

- เปลี่ยนได้อิสระ ไม่กระทบข้อมูลสินค้า
- สินค้า productType = 'service' ยังอยู่ แต่จะไม่ popup ถ้าเป็น RETAIL
- Service Charge จะถูกปิดอัตโนมัติเมื่อเปลี่ยนเป็น RETAIL

## 11. สรุป

| สิ่งที่เลือก | คำถาม | ตัวเลือก |
|-------------|-------|---------|
| Business Type | "ธุรกิจของคุณเป็นแบบไหน?" | SERVICE / RETAIL |
| Business Scale | "ขนาดธุรกิจ?" | BUSINESS (1 ร้าน) / ENTERPRISE (หลายสาขา) |

ทั้ง 2 มิติเลือกอิสระจากกัน ทำให้รองรับได้ทุกรูปแบบธุรกิจ ตั้งแต่ร้านตัดผมเล็กๆ 1 ร้าน ไปจนถึงเครือร้านสะดวกซื้อขนาดใหญ่หลายสิบสาขา
