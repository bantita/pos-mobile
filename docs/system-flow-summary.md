## 2. โครงสร้างร้านค้า

### 2 มิติ: ประเภท × ขนาด

| | BUSINESS (ร้านเดียว) | ENTERPRISE (หลายสาขา) |
|---|---|---|
| **SERVICE** (บริการ) | ร้านตัดผม 1 ร้าน | เครือสปา 5 สาขา |

```

```

## 3. M01 Authentication

### Flow สมัคร

```
Welcome → กรอกเบอร์โทร → OTP (123456)
  → เลือกประเภท (SERVICE/RETAIL)
  → เลือกขนาด (BUSINESS/ENTERPRISE)
  → ข้อมูลร้าน (ชื่อ, ที่อยู่, Tax ID)
```

```
กรอกเบอร์ → OTP → เข้าระบบ
```

## 5. M03 POS Sale (ขายสินค้า)

### 5.1 Flow หลัก

```
เปิดกะ (กรอกเงินเปิดกะ)
  │
  ▼
หน้า POS (3 โหมด: Button / Scan / Both)
  │
  ├── [RETAIL] กดสินค้า → เพิ่มตะกร้า
  │     └── ถ้าหลาย UOM → Popup เลือกหน่วย
  │
  ├── [SERVICE] กดสินค้าบริการ → Popup เลือกช่าง → เพิ่มตะกร้า
  │     └── กดซ้ำ → เลือกช่างคนอื่นได้
  │
  ├── สแกนบาร์โค้ด → เพิ่มอัตโนมัติ
  │
  ▼
ตะกร้า (Cart)
  │
  ├── แก้ไขจำนวน / ลบ / ส่วนลดรายสินค้า
  ├── ส่วนลดท้ายบิล (จำนวน / %)
  ├── ใช้คูปอง
  ├── เลือกสมาชิก CRM (สะสมแต้ม)
  │
  ▼
ชำระเงิน
  │
  ├── เงินสด (กรอกรับ → คำนวณทอน)
  ├── QR Code (แสดง QR)
  ├── บัตรเครดิต
  ├── โอนเงิน
  ├── E-Wallet
  ├── แยกจ่ายหลายช่องทาง (Split Payment)
  │
  ▼
พิมพ์ใบเสร็จ → บิลสำเร็จ
  │
  ├── ตัดสต๊อก (local + queue ERP)
  ├── สะสมแต้ม CRM
  ├── บันทึกยอดกะ
  │
  ▼
กลับหน้า POS (ขายต่อ)
```

### 5.2 ฟีเจอร์เสริม

| ฟีเจอร์ | คำอธิบาย |
|---------|----------|
| พักบิล (Hold) | เก็บบิลไว้ก่อน → เรียกคืนทีหลัง |
| ยกเลิกบิล (Void) | ต้องมีเหตุผล + อนุมัติ (Manager+) |
| Kiosk Mode | ล็อกหน้าจอ PIN ปลดล็อก |
| Customer Display | หน้าจอที่ 2 แสดงรายการให้ลูกค้า |
| เงินเข้า/ออก (Cash In/Out) | ระหว่างวัน + บันทึกเหตุผล |

### 5.3 Flow กะ (Shift)

```
เปิดกะ → กรอกเงินเปิดกะ
  │
  ├── ขายสินค้า (ยอดสะสม)
  ├── เงินเข้า (+)
  ├── เงินออก (-)
  │
  ▼
ปิดกะ → นับเงินจริง → เปรียบเทียบ (ควรเป็น vs นับได้) → ผลต่าง
```

## 6. M04 สินค้า & กำหนดราคา

### 6.1 จัดการสินค้า

```
รายการสินค้า (ค้นหา/กรอง)
  │
  ├── เพิ่มสินค้าใหม่
  │     ├── ข้อมูลพื้นฐาน (รหัส, บาร์โค้ด, ชื่อ)
  │     ├── หมวดหมู่ & Brand
  │     ├── หน่วยนับ (UOM) — หลายหน่วย + หลายบาร์โค้ด
  │     ├── ราคาทุน / ราคาขาย / Margin preview
  │     ├── VAT% (0% / 7%)
  │     ├── ประเภท (สินค้าทั่วไป / สินค้าบริการ)
  │     ├── สต๊อกขั้นต่ำ
  │     └── รูปภาพ
  │
  ├── แก้ไขสินค้า (Audit log ราคา)
  ├── ลบสินค้า
  └── Import สินค้า (Excel/CSV)

```

### 6.2 กำหนดราคา (Enterprise)

```
รายการเอกสารกำหนดราคา
  │
  ├── [สร้างเอกสารใหม่]
  │     Step 1: เลือกโหมด
  │       ├── ทุกสาขาราคาเดียวกัน
  │       └── เฉพาะสาขา → เลือกสาขา
  │     Step 2: เลือกสินค้า (checkbox + ค้นหา)
  │     Step 3: กำหนดราคา
  │       ├── ราคากลาง (แสดง)
  │       ├── ราคาเก่า (แสดง)
  │       ├── ราคาใหม่ (กรอก)
  │       └── ผลต่าง (+/-)
  │     → ยืนยันเอกสาร
  │
  ├── [Import Excel] — อ่านไฟล์ปรับราคา
  ├── [Export Excel] — ส่งออกราคาปัจจุบัน
  └── [Download Template] — ไฟล์ตัวอย่าง
```

## 7. M05 คลังสินค้า & ERP Sync

### 7.1 Flow คลัง

```
┌─ รับสินค้า (Receive) ─────────────────────────┐
│ สร้างเอกสาร → เลือก Supplier                   │
│ → เพิ่มสินค้า (สแกน/เลือก) → ยืนยัน            │
│ → สต๊อกเพิ่ม + Sync ERP                        │
└─────────────────────────────────────────────────┘

┌─ เบิกสินค้า (Issue) ──────────────────────────┐
│ สร้างเอกสาร → เลือกจุดขาย/แผนก                │
│ → เลือกสินค้า + จำนวน → ยืนยัน                 │
│ → สต๊อกลด + Sync ERP                          │
└─────────────────────────────────────────────────┘

┌─ โอนข้ามสาขา (Transfer) — ENTERPRISE ─────────┐
│ เลือกสาขาต้นทาง → สาขาปลายทาง                  │
│ → เลือกสินค้า → ยืนยัน                          │
│ → ต้นทางลด / ปลายทางเพิ่ม + Sync ERP           │
└─────────────────────────────────────────────────┘

┌─ นับสต๊อก (Stock Count) ──────────────────────┐
│ สร้างรอบนับ → เลือกหมวด/ทั้งหมด                │
│ → กรอก/สแกนจำนวนจริง → เปรียบเทียบ             │
│ → ยืนยันปรับสต๊อก + Sync ERP                   │
└─────────────────────────────────────────────────┘
```

### 7.2 ERP Sync

```
POS (Local Stock) ←── Sync ──→ ERP (Central Stock)

Online:  ขาย → ตัด local → API ทันที → ERP ตัด
Offline: ขาย → ตัด local → Queue → Online → Batch Sync
```

## 8. M06 CRM สมาชิก

### 8.1 เมนู CRM ทั้งหมด

```
CRM Sidebar:
  ├── ข้อมูลสมาชิก (Listing + กดดูรายละเอียด)
  ├── เพิ่มสมาชิก (ฟอร์ม)
  ├── ระดับสมาชิก (ตั้งค่าเงื่อนไข)
  ├── ตั้งค่าคะแนน (อัตราได้/แลก/หมดอายุ/โบนัส)
  ├── ปรับปรุงคะแนน (Manual +/- แต้ม)
  ├── คูปอง / Voucher (สร้าง/จัดการ)
  ├── Campaign Marketing (LINE/SMS/Email)
  ├── Gamification (วงล้อ/Lucky Box/Check-in)
  ├── Segment ลูกค้า (แบ่งกลุ่มอัตโนมัติ)
  ├── ประวัติการซื้อ
  ├── Wallet / Credit
  ├── รายงาน CRM
  └── ตั้งค่า CRM (LINE OA + e-Card + แจ้งเตือน)
```

### 8.2 Flow สะสมแต้ม (ที่ POS)

```
ขายสินค้า → เลือกสมาชิก (สแกน/ค้นหา)
  │
  ▼
ชำระเงิน → บิลสำเร็จ
  │
  ▼
คำนวณแต้ม: ยอด ÷ อัตรา (เช่น 25 บาท = 1 แต้ม)
  × ตัวคูณระดับ (เช่น Gold = x1.5)
  × ตัวคูณวันเกิด (ถ้าตรง = x2)
  │
  ▼
แจ้งเตือนลูกค้า (LINE): "ได้รับ 50 แต้ม คงเหลือ 1,300"
  │
  ▼
ตรวจเงื่อนไขอัปเกรดระดับ
  ├── ถึงเงื่อนไข → อัปเกรด + แจ้ง LINE
  └── ไม่ถึง → ไม่ทำอะไร
```

### 8.3 Flow ใช้คูปอง (ที่ POS)

```
ตะกร้ามีสินค้า → กดใช้คูปอง
  │
  ▼
กรอกรหัสคูปอง / สแกน QR
  │
  ▼
ตรวจสอบ:
  ├── คูปองหมดอายุ? → ❌ แจ้ง
  ├── ใช้ครบจำนวนแล้ว? → ❌ แจ้ง
  ├── เงื่อนไขตรง? (ระดับสมาชิก, ยอดขั้นต่ำ)
  │     └── ไม่ตรง → ❌ แจ้ง
  └── ✅ ใช้ได้ → คำนวณส่วนลด → แสดงในบิล
```

### 8.4 Flow ระดับสมาชิก

```
Member → Silver → Gold → Platinum → VIP

เงื่อนไขอัปเกรด (แต่ละระดับ):
  - ยอดซื้อสะสม >= X บาท
  - จำนวนบิล >= Y บิล
  - ภายใน Z วัน

สิทธิ์ที่ได้:
  - ส่วนลด (%)
  - อัตราแต้มคูณ (x multiplier)
  - คูปองพิเศษ
  - วันหมดอายุระดับ (ต้องรักษายอด)
```

## 9. M07 โปรโมชั่น

### 9.1 Promotion Engine (เอนจินคำนวณโปรโมชั่น)

```
สินค้าในตะกร้า
  │
  ▼
┌─ Promotion Engine ─────────────────────────────┐
│ 1. ดึงโปรโมชั่นทั้งหมดที่ active               │
│ 2. ตรวจเงื่อนไข (Conditions):                  │
│    ├── วัน/เวลา (จันทร์-ศุกร์, 10:00-14:00)   │
│    ├── สมาชิก (ระดับ Gold+)                    │
│    ├── สินค้า (เฉพาะหมวด X)                    │
│    ├── ยอดขั้นต่ำ (>= 500 บาท)                │
│    ├── คูปอง (ต้องใส่รหัส)                     │
│    ├── สาขา (เฉพาะสาขา Y)                     │
│    └── สินค้ายกเว้น (ไม่รวมสินค้า Z)           │
│                                                 │
│ 3. คำนวณสิทธิ์ (Benefits):                     │
│    ├── ส่วนลด % (เช่น ลด 20%)                 │
│    ├── ส่วนลดจำนวนเงิน (เช่น ลด 50 บาท)      │
│    ├── ส่วนลดต่อชิ้น (เช่น ลด 5 บาท/ชิ้น)    │
│    ├── ของแถม (ซื้อ 2 แถม 1)                   │
│    └── ลดท้ายบิล (Bill Discount)               │
│                                                 │
│ 4. จัดการขัดแย้ง (Conflict Resolution):        │
│    ├── Best Benefit (เลือกดีสุด)               │
│    ├── Highest Priority (ลำดับก่อน)            │
│    └── First Match (ตัวแรกที่ตรง)              │
│                                                 │
│ 5. Audit Trail (บันทึกว่าใช้โปรฯ ไหน)         │
└─────────────────────────────────────────────────┘
  │
  ▼
แสดงส่วนลดในตะกร้า / ใบเสร็จ
```

### 9.2 ประเภทโปรโมชั่น

| ประเภท | ตัวอย่าง |
|--------|----------|
| ส่วนลด % | ลด 20% ทุกสินค้าหมวดเครื่องดื่ม |
| ส่วนลดจำนวนเงิน | ลด 100 บาท เมื่อซื้อครบ 500 |
| ซื้อ X แถม Y | ซื้อ 2 แถม 1 (สินค้าเดียวกัน) |
| Bundle | ซื้อ A+B+C ราคาพิเศษ 199 |
| คูปอง | กรอกรหัส SAVE20 ลด 20% |
| สมาชิก | Gold+ ลด 5% ทุกบิล |
| Flash Sale | 12:00-13:00 ลด 30% |

### 9.3 Flow สร้างโปรโมชั่น

```
เลือกประเภทโปร → ตั้งชื่อ/วันเริ่ม-สิ้นสุด
  → กำหนดเงื่อนไข (Conditions)
  → กำหนดสิทธิ์ (Benefits)
  → กำหนด Priority + Conflict Strategy
  → เลือกสาขา (ทุกสาขา / เฉพาะสาขา)
  → Preview & Activate
```

## 10. M08 รายงาน

### 10.1 รายงานที่มี

| หมวด | รายงาน |
|------|--------|
| **ขาย** | ยอดขายรายวัน, รายเดือน, เปรียบเทียบ |
| **ขาย** | ยอดขายตามช่องทางชำระ |
| **ขาย** | ยอดขายตามพนักงาน/ช่าง |
| **ขาย** | สินค้าขายดี / ขายไม่ดี |
| **กำไร** | กำไรขั้นต้น, Margin % |
| **สต๊อก** | สต๊อกคงเหลือ, สินค้าใกล้หมด |
| **สต๊อก** | รายงานรับ-เบิก-โอน |
| **CRM** | สมาชิกใหม่, Retention Rate |
| **CRM** | รายงานแต้ม (ได้/ใช้/หมดอายุ) |

| **Enterprise** | เปรียบเทียบสาขา |
| **Enterprise** | P&L ต่อสาขา |

### 10.2 สรุปประจำวัน (Daily Summary)

```
เลือกวันที่ → แสดง:
  1. KPI รวม (ยอดขาย, บิล, เฉลี่ย, กำไร, คืน)
```

## 14. Flow รวมวันปกติ (End-to-End)

```
06:00  ───── เปิดร้าน ─────
         Login → Dashboard
         เปิดกะเช้า (เงิน ฿5,000)

06:00-15:00  ───── ขายกะเช้า ─────
         กดสินค้า / สแกน → ตะกร้า → ชำระ → พิมพ์
         (ซ้ำ 80 บิล)
         ├── สมาชิกสะสมแต้ม (CRM)
         ├── ใช้คูปอง (Promotion Engine ตรวจ)
         ├── โปรฯ auto apply (ลด 20% เครื่องดื่ม)
         ├── สต๊อกตัดอัตโนมัติ → Queue ERP
         └── 09:30 เงินเข้า +฿5,000

12:00  ───── Auto Sync ─────
         Sync Queue → ERP (สต๊อก + ยอด)
         ERP → POS (สต๊อกล่าสุด + ราคาใหม่)

14:15  เงินออก -฿3,000 (นำฝากธนาคาร)

15:00  ───── เปลี่ยนกะ ─────
         ปิดกะเช้า (นับเงิน → ผลต่าง +฿50)
         เปิดกะบ่าย

15:00-22:00  ───── ขายกะบ่าย ─────
         (ซ้ำ 76 บิล)
         ├── สมาชิกใหม่ 3 คน
         ├── อัปเกรด Silver → Gold 1 คน
         └── 18:00 เงินออก -฿2,000

22:00  ───── ปิดร้าน ─────
         ปิดกะบ่าย (นับเงิน → ผลต่าง -฿20)
         ดูสรุปประจำวัน:
           ยอดขาย ฿48,320 | 156 บิล | กำไร ฿12,080
           สต๊อกหมด 1 | ใกล้หมด 3
           แต้มแจก 1,250 | สมาชิกใหม่ 3
         Final Sync → ERP
         Export PDF สรุป
         Logout
```
